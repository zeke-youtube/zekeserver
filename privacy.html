🛡️ Zekebot Privacy Policy

This document explains how Zekebot handles user data and behavior within the Zeke Server environment.

1. What Zekebot Collects

  • Basic message content used to detect filtered words and trigger commands.
  • User IDs and strike counts for moderation purposes (stored temporarily and not shared externally).
  • Command logs for bot usage statistics and debugging — optional and local to server admins.

2. What Zekebot Does Not Collect

  • Does not access your private messages (unless sent to Zekebot directly).
  • Does not store personal information such as name, email, or location.
  • Does not share any user behavior or data with third parties.

3. How Data is Used

  • Automod systems use message content to detect violations (e.g. filtered language).
  • Strike counters help Zekebot enforce moderation policies fairly.
  • DM notifications ensure violations are privately addressed.

4. Data Retention

  • Strike data and logs are stored in memory during bot runtime only.
  • When Zekebot restarts, all temporary data is reset unless persistent storage is manually configured by server admins.

5. Consent

  • By using Zekebot and interacting on Zeke Server, you accept this Privacy Policy.
  • Updates will be posted here. Continued use implies consent.
